# 运营监控

本文说明 BuildingAI 的健康检查、日志、队列、PM2 和日常巡检方式。

## 健康检查

基础健康检查：

```text
GET /consoleapi/health
```

详细健康检查：

```text
GET /consoleapi/health/detail
```

数据库与 Redis：

```text
GET /consoleapi/health/db
```

健康检查模块基于 `@nestjs/terminus`，检查内容包括应用状态、HTTP ping、heap、RSS、磁盘、数据库和 Redis。

## Docker healthcheck

Docker Compose 中的 Node 服务会检查：

1. `/consoleapi/health` 是否可访问。
2. PM2 进程是否在线。

查看容器状态：

```bash
docker compose ps
docker compose logs -f nodejs
```

## PM2 运维

常用命令：

```bash
pnpm pm2:status
pnpm pm2:logs
pnpm pm2:monitor
pnpm pm2:reload
pnpm pm2:restart
pnpm stop
pnpm pm2:save
pnpm pm2:flush
```

日志默认位置：

```text
logs/pm2/api-out.log
logs/pm2/api-error.log
logs/pm2/api-combined.log
```

## 应用日志

应用日志由自定义 Logger 输出，可通过 `.env` 控制：

```dotenv
LOG_LEVELS=error,warn,debug,fatal
LOG_WRITE_LEVELS=error,warn,debug,fatal,log
LOG_TO_FILE=true
LOG_CLEAN_INTERVAL=3
```

建议生产环境开启文件日志，并定期清理或归档。

## 队列监控

队列基于 BullMQ + Redis。

默认队列：

- `default`
- `email`
- `vectorization`

重点关注：

- 等待任务数。
- 失败任务数。
- 向量化任务耗时。
- Redis 连接状态。
- Worker 是否运行。

知识库文档长时间处理中时，优先检查 `vectorization` 队列。

## 数据库监控

关注指标：

- PostgreSQL 连接是否正常。
- 数据目录磁盘空间。
- 慢查询。
- 数据库扩展是否可用。
- 备份是否成功。

推荐定期备份：

```bash
pg_dump -h <host> -U <user> -d <database> > buildingai.sql
```

## Redis 监控

关注指标：

- Redis 是否可连接。
- 内存使用。
- 队列积压。
- 是否配置密码。
- 是否被公网暴露。

生产环境不要公网暴露 Redis。

## 日常巡检清单

- `/consoleapi/health` 是否正常。
- PM2 进程是否 online。
- Docker 容器是否 healthy。
- `logs/pm2/api-error.log` 是否有新增异常。
- PostgreSQL 和 Redis 是否可用。
- `vectorization` 队列是否积压。
- 磁盘空间是否充足。
- 模型 Provider 和 Secret 是否可用。
- 最近发布后是否有 401、403、500 等异常增加。

## 发布后验证

1. 访问 `/consoleapi/health`。
2. 登录后台 `/console/dashboard`。
3. 打开 `/console/provider` 检查模型配置。
4. 创建一次普通聊天。
5. 打开一个 Agent 进行流式对话。
6. 打开一个知识库进行问答。
7. 查看 PM2 和应用错误日志。
