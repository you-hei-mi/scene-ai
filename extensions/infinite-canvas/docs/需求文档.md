# Infinite Canvas 需求文档

## 1. 项目定位

Infinite Canvas 是 BuildingAI 的原生应用扩展，目标是在 `extensions/infinite-canvas` 中复刻 `D:\myGit\Infinite-Canvas-main` 的无限画布、多模型聚合、资产库、提示词库、工作流管理和外部工具协同能力。

复刻原则：

- 功能、页面结构、核心交互参考原项目 `需求文档.md` 与源码。
- 前端框架、主题风格、路由、HTTP 客户端、UI 组件、鉴权上下文与当前 BuildingAI 系统保持一致。
- 当前系统已有能力优先复用，包括扩展 SDK、TypeORM 扩展 schema、系统上传存储、AI SDK、公共 UI 与通知体系。
- 当前系统缺失但原项目要求的能力按阶段补齐，并保留原项目 JSON/workflow 数据的导入导出兼容。

## 2. 用户与使用场景

| 用户 | 主要场景 | 成功标准 |
| --- | --- | --- |
| AIGC 创作者 | 在画布中组合提示词、参考图、图片/视频/LLM 节点并生成内容 | 能创建画布、拖拽节点、连线、保存和恢复工作 |
| 设计师/运营 | 管理本地素材、分类、标签，并拖入画布复用 | 资产库可上传、搜索、分类、移动、加入画布 |
| AI 工程师 | 接入 ComfyUI、RunningHub、ModelScope、自定义 Provider | Provider/工作流可配置、可测试、可用于节点 |
| 教学/培训用户 | 复用模板和演示多模型对比 | 画布可导入导出，提示词库可沉淀模板 |

## 3. 范围与版本边界

### V1：画布闭环

- 扩展骨架和路由接入。
- 启动器首页、画布列表、经典画布、智能画布、资产库、提示词库、Provider 设置、ComfyUI 设置基础页面。
- 画布支持节点创建、拖拽、框选、多选、连线、右键菜单、撤销/重做、缩放/平移、自动保存。
- 首批节点类型：`image`、`prompt`、`generator`、`video`、`loop`、`llm`、`output`、`comfy`、`rh`、`ltxDirector`。
- 支持原项目 canvas JSON 导入/导出，保持 `nodes[]`、`edges[]`、`viewport`、`settings` 兼容。
- 资产库与提示词库基础 CRUD。

### V2：AI 与任务能力

- Provider 配置与模型列表管理。
- 文本、图片、视频、工作流任务统一进入 `GenerationTask`。
- 优先复用 BuildingAI AI SDK；缺失协议建立扩展适配层。
- 无凭证或外部服务不可用时，界面显示未配置/不可用状态，不影响画布编辑。

### V3：完整生态

- RunningHub/ComfyUI 工作流双向管理。
- 共享文件夹、智能分类、头像注册、媒体预览、视频临时托管。
- 复刻原项目独立工具页：`online`、`enhance`、`zimage`、`klein`、`angle`、`gpt-chat`。
- Chrome/Photoshop 插件协同优先保留协议兼容接口，是否迁移插件源码另行评估。

## 4. 页面需求

| 路由 | 页面 | 需求 |
| --- | --- | --- |
| `/extension/infinite-canvas/` | 工作台首页 | 显示入口卡片、最近画布、Provider 状态、资产统计 |
| `/extension/infinite-canvas/canvas-list` | 画布管理 | 新建、搜索、分页、软删除、恢复、永久删除、导入导出 |
| `/extension/infinite-canvas/canvas/:id` | 经典画布 | 节点编排、连线、撤销/重做、保存、资源面板 |
| `/extension/infinite-canvas/smart-canvas/:id` | 智能画布 | 群组组织、循环级联、模板注入、增强撤销 |
| `/extension/infinite-canvas/asset-manager` | 资产库 | 库/分类/素材管理、标签、批量移动、加入画布 |
| `/extension/infinite-canvas/api-settings` | Provider 设置 | Provider CRUD、模型配置、连接测试、密钥屏蔽 |
| `/extension/infinite-canvas/comfyui-settings` | ComfyUI 设置 | 实例管理、连通测试、工作流入口 |
| `/extension/infinite-canvas/console/*` | 后台管理 | 系统化管理入口，复用 BuildingAI console layout |

## 5. 数据模型

核心数据使用扩展独立数据库 schema，媒体文件使用当前系统上传存储；同时保留原项目 JSON 兼容。

| 模型 | 说明 |
| --- | --- |
| `CanvasProject` | 项目/画布分组 |
| `Canvas` | 画布元信息、节点、连线、视口、设置、软删除 |
| `CanvasNode` | 逻辑节点结构，V1 存储在 `Canvas.nodes` JSON 内 |
| `CanvasEdge` | 逻辑连线结构，V1 存储在 `Canvas.edges` JSON 内 |
| `AssetLibrary` / `AssetCategory` / `AssetItem` | 资产库、分类、素材 |
| `PromptLibrary` / `PromptCategory` / `PromptItem` | 提示词库、分类、模板 |
| `AiProviderProfile` | 扩展级 Provider 配置快照 |
| `WorkflowPreset` | ComfyUI、RunningHub、Canvas workflow 预设 |
| `GenerationTask` | 图片、视频、LLM、工作流任务状态 |

## 6. API 需求

扩展 Web API 前缀：`/infinite-canvas/api/...`。

扩展后台 API 前缀：`/infinite-canvas/consoleapi/...`。

首版 API 域：

- `projects`：项目列表、新建、更新、删除。
- `canvases`：列表、新建、详情、保存、软删除、恢复、永久删除、导入导出。
- `assets`：资产库、分类、素材基础 CRUD。
- `prompts`：提示词库、分类、提示词基础 CRUD。
- `providers`：Provider 配置、模型列表、连接测试占位。
- `workflows`：workflow 预设 CRUD、原项目 workflow JSON 导入导出。
- `tasks`：生成任务创建、状态查询、失败信息展示。

## 7. 交互与体验

- 画布为第一屏核心体验，不做营销式落地页。
- 使用 BuildingAI UI 的按钮、输入框、表格、弹窗、菜单、通知与图标。
- 视觉优先级：BuildingAI 系统风格优先；原项目作为布局、信息架构和交互能力参照。
- 画布区域需要稳定尺寸，节点拖动、选中、连线和缩放不得造成布局跳动。
- 无外部服务配置时，功能入口保持可见但显示清晰的未配置状态。

## 8. 验收标准

- 能从主应用进入 Infinite Canvas 扩展首页。
- 能新建画布，进入画布，添加首批节点类型，拖动节点并保存。
- 刷新页面后画布节点、连线、视口恢复。
- 能导出原项目兼容 JSON，再导入成新画布。
- 能创建资产库/提示词库条目，并从画布侧边面板查看。
- Provider/ComfyUI/Workflow/Task 页面有可用的基础管理界面和未配置提示。
- `pnpm --filter infinite-canvas build:web`、`pnpm --filter infinite-canvas build:api`、相关测试通过或记录明确失败原因。

## 9. 风险与限制

- 原项目为 FastAPI + 静态多页，当前实现为 BuildingAI 原生扩展，不能逐文件照搬运行时。
- 原项目授权包含署名和用途限制，文档与界面中需保留来源说明，不规划商业化发布。
- 即梦 CLI、Codex CLI、RunningHub、ComfyUI 等外部能力受用户本机环境和凭证影响，必须做可用性检测与降级提示。
- V1 将节点和连线以 JSON 保存到 `Canvas`，后续如需大规模查询再拆分独立表。
