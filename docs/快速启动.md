# 快速启动

本文用于从零启动 BuildingAI 开发或调试环境。

## 环境要求

- Node.js: `>=22.20.x <23`
- pnpm: `10.20.0`
- Docker 与 Docker Compose: 推荐用于快速启动依赖服务和整套应用
- PostgreSQL: 推荐 17.x
- Redis: 推荐 8.x
- 最低资源建议: 2 核 CPU、4 GB 内存、5 GB 可用磁盘空间

## Docker 快速启动

```bash
cp .env.example .env
pnpm docker:up
```

Docker Compose 会启动 `redis`、`postgres`、`nodejs` 三个服务。默认应用端口为 `4090`，启动完成后访问：

```text
http://localhost:4090/install
```

首次访问 `/install` 会进入初始化安装流程。

## 本地一键启动

```bash
cp .env.example .env
pnpm start
```

`pnpm start` 会执行以下流程：

1. 兼容旧扩展构建产物。
2. 安装依赖。
3. 同步 `.env` 配置。
4. 进入 CLI setup 流程。
5. 构建并通过 PM2 启动 API 服务。

## 开发模式

```bash
pnpm dev
```

也可以按模块分别启动：

```bash
pnpm dev:api
pnpm dev:web
```

常用构建命令：

```bash
pnpm build
pnpm build:api
pnpm build:web
```

常用检查命令：

```bash
pnpm lint
pnpm typecheck
```

## 访问地址

- 安装初始化: `http://localhost:4090/install`
- Web 站点/API 统一入口: `http://localhost:4090`
- 健康检查: `http://localhost:4090/consoleapi/health`
- 前端开发服务默认端口: `4091`

## 常见问题

- 如果 `.env` 缺少新配置，运行 `pnpm sync-env`。
- 如果 API 构建产物缺失，运行 `pnpm build:api`。
- 如果 PM2 进程异常，运行 `pnpm pm2:status` 和 `pnpm pm2:logs`。
- 如果 Docker 启动慢，优先查看 `buildingai-nodejs` 容器日志。
