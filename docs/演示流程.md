# 演示流程

本文用于产品演示、售前演示和功能验收。

## 1. 初始化安装

访问：

```text
http://localhost:4090/install
```

演示要点：

- 欢迎页。
- 创建管理员账号。
- 配置站点名称、描述、Logo、图标和主题。
- 初始化成功后进入系统。

关键路径：

- `/install`
- `packages/client/src/pages/install/index.tsx`

## 2. 登录与控制台概览

访问：

```text
http://localhost:4090/login
```

登录后进入：

```text
/console/dashboard
```

演示要点：

- 控制台统一挂载在 `/console/*`。
- 控制台菜单和权限来自当前登录用户。
- 管理员可进入系统配置、用户、权限、模型、知识库、Agent 和运营模块。

## 3. 配置模型与服务商

访问：

```text
/console/provider
```

演示要点：

- 新增或启用 AI 服务商。
- 配置模型。
- 进入 `/console/secret` 管理密钥。
- 进入 `/console/mcp` 管理 MCP 服务和工具。

## 4. 创建和配置 Agent

访问：

```text
/agents/workspace
```

或：

```text
/agents
```

演示步骤：

1. 创建 Agent。
2. 选择直接创建、Coze 或 Dify。
3. 进入 `/agents/:id/configuration`。
4. 配置角色提示词。
5. 配置表单变量。
6. 绑定知识库。
7. 配置上下文、引用、MCP 工具、联网搜索和文件上传。
8. 配置开场白、推荐问题、自动追问、快捷指令和聊天头像。
9. 配置模型路由和语音能力。
10. 使用右侧调试预览验证效果。

## 5. Agent 聊天体验

访问：

```text
/agents/:id/chat
```

演示要点：

- 发送问题并展示流式回复。
- 演示历史会话 `/agents/:id/c/:uuid`。
- 演示重新生成、编辑消息、点赞/点踩。
- 演示语音输入、语音朗读和文件上传。
- 如果已绑定知识库，展示引用和知识库增强回答。

## 6. Agent 发布

访问：

```text
/agents/:id/publish
```

演示要点：

- 开启网页访问。
- 复制公开链接。
- 打开 `/agents/:agentId/:accessToken`。
- 展示 iframe 嵌入代码。
- 展示浮窗脚本。
- 如需广场发布，演示发布到广场和后台审核流程。

## 7. 知识库创建与问答

访问：

```text
/datasets
```

演示步骤：

1. 创建或打开知识库 `/datasets/:id`。
2. 上传文档或从 URL 导入。
3. 查看文档列表、分类 tab、预览与批量操作。
4. 等待解析和向量化完成。
5. 在右侧问答面板提问。
6. 展示知识库回答、引用片段和检索效果。
7. 演示成员管理、标签编辑、文档转移和发布到知识广场。

## 8. 控制台审核与运营

常用入口：

- Agent 审核: `/console/agent`
- 知识库审核: `/console/datasets`
- 聊天记录: `/console/chat/record`
- 聊天配置: `/console/chat/config`
- 模型服务商: `/console/provider`
- MCP 管理: `/console/mcp`
- 密钥管理: `/console/secret`
- 扩展管理: `/console/extension`
- 前台装修: `/console/decorate/agents`、`/console/decorate/apps`、`/console/decorate/layout`
- 系统配置: `/console/system/website-config`、`/console/system/storage-config`、`/console/system/pay-config`
- 权限配置: `/console/access/menu`、`/console/access/permission`、`/console/access/role`

## 演示前检查

- 服务健康检查通过: `/consoleapi/health`。
- 已完成初始化安装。
- 已配置可用 Provider、模型和密钥。
- 如果演示知识库，至少准备一个已完成向量化的文档。
- 如果演示发布链接，确认 `APP_DOMAIN` 或本地访问地址正确。
