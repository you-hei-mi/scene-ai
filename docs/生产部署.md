# 生产部署

本文说明 BuildingAI 的生产部署建议。推荐使用 Docker Compose 部署，PM2 可用于 Node 服务进程管理。

## 推荐方式

```bash
cp .env.example .env
pnpm docker:up
```

Docker Compose 包含：

- `redis`: Redis 8.2.2。
- `postgres`: PostgreSQL 17.6。
- `nodejs`: Node 22.20.0，执行应用启动流程。

默认数据卷：

- Redis: `docker/data/redis`
- PostgreSQL: `docker/data/postgres/postgres_data`

## 生产必改配置

编辑 `.env`：

```dotenv
APP_DOMAIN=https://your-domain.example.com
JWT_SECRET=replace-with-a-strong-secret
DB_PASSWORD=replace-with-a-strong-db-password
REDIS_PASSWORD=replace-with-a-strong-redis-password
SERVER_CORS_ORIGIN=https://your-domain.example.com
SERVER_SHOW_DETAILED_ERRORS=false
DB_SYNCHRONIZE=false
LOG_TO_FILE=true
```

如果前后端使用不同域名，配置：

```dotenv
VITE_PRODUCTION_APP_BASE_URL=https://api.your-domain.example.com
```

## Docker 资源限制

可通过 `.env` 调整：

```dotenv
DOCKER_MEMORY_LIMIT=6144M
DOCKER_CPU_LIMIT=2.0
DOCKER_MEMORY_RESERVATION=512M
```

生产环境建议根据机器规格设置合理限制，避免 Node 构建或运行时耗尽资源。

## 端口

默认应用端口：

```dotenv
SERVER_PORT=4090
```

Docker 会将宿主机 `SERVER_PORT` 映射到容器内同名端口。

PostgreSQL 和 Redis 默认不固定暴露宿主机端口。如确需调试，可设置：

```dotenv
POSTGRES_EXTERNAL_PORT=5432
REDIS_EXTERNAL_PORT=6379
```

生产环境不建议公网暴露 `5432` 和 `6379`。

## PM2 管理

API 生产入口是：

```text
packages/api/dist/main.js
```

常用命令：

```bash
pnpm pm2:start
pnpm pm2:status
pnpm pm2:logs
pnpm pm2:reload
pnpm pm2:restart
pnpm stop
pnpm pm2:save
pnpm pm2:flush
```

`reload` 适合零停机重载，`restart` 适合配置或依赖发生明显变化后的重启。

## 健康检查

应用健康检查：

```text
/consoleapi/health
```

详细检查：

```text
/consoleapi/health/detail
/consoleapi/health/db
```

Docker healthcheck 会优先请求 `/consoleapi/health`，并兜底检查 PM2 进程在线状态。

## 发布流程

推荐流程：

```bash
git pull
pnpm install
pnpm sync-env
pnpm build
pnpm pm2:reload
pnpm pm2:status
```

Docker 部署可使用：

```bash
git pull
docker compose up -d --build
docker compose logs -f nodejs
```

## 回滚建议

1. 保留上一个稳定 Git tag 或 commit。
2. 数据库升级前备份 PostgreSQL 数据。
3. 回滚代码后重新构建。
4. 使用 `pnpm pm2:restart` 或重新启动 Docker 服务。

## 安全建议

- 生产环境必须替换 `JWT_SECRET`。
- 不要使用默认数据库密码。
- 不要公网暴露 Redis。
- 后台管理入口建议放在 HTTPS 后。
- 定期备份 `docker/data/postgres/postgres_data`。
- 定期检查 `logs/` 和磁盘空间。
