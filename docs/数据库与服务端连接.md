# 数据库与服务端连接

本文说明“虚拟机部署调试环境，本地电脑开发代码，实时调试和提交代码”的推荐方式。

## 推荐拓扑

推荐调试环境：

- 虚拟机运行 PostgreSQL、Redis 和可选 API 服务。
- 本地电脑保留源码、编辑器、Git 和开发命令。
- 本地通过 `.env` 连接虚拟机数据库和 Redis。
- 代码提交在本地完成，虚拟机通过 Git 拉取后构建和重启。

## 虚拟机端口

调试环境通常需要开放：

| 端口 | 服务 | 用途 |
| --- | --- | --- |
| `4090` | BuildingAI API | 本地前端或浏览器访问虚拟机服务 |
| `5432` | PostgreSQL | 本地 API 连接虚拟机数据库 |
| `6379` | Redis | 本地 API 连接虚拟机 Redis |

安全提示：

- 以上端口只建议在内网、VPN 或安全组白名单中开放。
- 生产环境不要公网裸露 `5432` 和 `6379`。
- Redis 必须设置密码。
- PostgreSQL 必须使用强密码。

## 模式一：本地运行 API，连接虚拟机数据库和 Redis

适合日常后端开发和断点调试。

本地 `.env` 示例：

```dotenv
SERVER_PORT=4090

DB_TYPE=postgres
DB_HOST=<虚拟机内网IP>
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=<虚拟机数据库密码>
DB_DATABASE=buildingai
DB_DEV_SYNCHRONIZE=true

REDIS_HOST=<虚拟机内网IP>
REDIS_PORT=6379
REDIS_PASSWORD=<虚拟机Redis密码>
REDIS_DB=0

VITE_DEVELOP_APP_BASE_URL=http://localhost:4090
```

本地启动：

```bash
pnpm install
pnpm sync-env
pnpm dev:api
```

前端另开终端：

```bash
pnpm dev:web
```

浏览器访问前端开发服务，前端请求本地 API，本地 API 连接虚拟机数据库和 Redis。

## 模式二：虚拟机运行 API，本地只开发和提交

适合接近线上部署方式的调试。

虚拟机执行：

```bash
git pull
pnpm install
pnpm sync-env
pnpm build
pnpm pm2:restart
pnpm pm2:logs
```

本地前端如果要连虚拟机 API，可配置：

```dotenv
VITE_DEVELOP_APP_BASE_URL=http://<虚拟机IP>:4090
```

然后本地启动：

```bash
pnpm dev:web
```

## Docker 调试部署

虚拟机使用 Docker Compose：

```bash
cp .env.example .env
pnpm docker:up
docker compose logs -f nodejs
```

如需让本地直连虚拟机 PostgreSQL 和 Redis，设置：

```dotenv
POSTGRES_EXTERNAL_PORT=5432
REDIS_EXTERNAL_PORT=6379
REDIS_PASSWORD=<强密码>
```

然后重启：

```bash
docker compose up -d
```

## 实时调试建议

后端调试优先使用模式一：

```bash
pnpm dev:api
```

优点：

- 本地修改代码后立即生效。
- 可以使用本地 IDE 断点。
- 仍然复用虚拟机数据库和 Redis 数据。

前端调试：

```bash
pnpm dev:web
```

如果前端需要连接虚拟机 API，将 `VITE_DEVELOP_APP_BASE_URL` 指向虚拟机 `4090`。

## 提交与部署流程

本地开发：

```bash
git status --short
pnpm typecheck
git add .
git commit -m "docs: 初始化项目文档"
git push
```

虚拟机部署：

```bash
git pull
pnpm install
pnpm sync-env
pnpm build
pnpm pm2:reload
pnpm pm2:status
```

如果使用 Docker：

```bash
git pull
docker compose up -d --build
docker compose logs -f nodejs
```

## 排查清单

- 本地能否 ping 通虚拟机 IP。
- 虚拟机安全组是否开放 `4090`、`5432`、`6379`。
- PostgreSQL 是否允许远程连接。
- Redis 是否设置了密码且允许远程连接。
- `.env` 中 `DB_HOST` 和 `REDIS_HOST` 是否为虚拟机 IP。
- 本地 API 日志是否出现数据库或 Redis 连接错误。
- 虚拟机 PM2 日志是否有启动异常。
- Docker 容器是否 healthy。
