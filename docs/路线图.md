# 路线图

本文记录项目近期、中期和长期建设方向，便于后续版本规划。

## 近期

- 补齐项目文档中心。
- 完成 `.env` 配置矩阵。
- 补充 Docker 和 PM2 部署排障指南。
- 完成虚拟机调试环境说明。
- 建立 `docs/todo` 版本开发计划机制。
- 梳理知识库治理、向量化失败处理和检索调优流程。
- 建立产品演示脚本，覆盖安装、模型、Agent、知识库和发布。

## 中期

- 增加扩展开发手册，覆盖 `pnpm extension:create`、`pnpm extension:release`、manifest、schema 和 seed。
- 增加 Skills 管理手册，覆盖 `pnpm skills sync/remove`。
- 完善运营监控指标面板，包括队列、PM2、健康检查、日志和磁盘。
- 增加关键后端模块测试覆盖。
- 增加前端核心流程的端到端验收清单。
- 治理当前中文注释和文档在终端中显示乱码的问题。
- 完善数据库升级、扩展升级和回滚流程。

## 长期

- 建立插件市场规范和扩展审核机制。
- 增加企业级审计日志和权限追踪。
- 建立自动化发布和版本回滚流水线。
- 建立知识库质量评估体系，包括召回率、引用质量、失败率和用户反馈。
- 增加多环境配置管理，覆盖开发、测试、预发和生产。
- 增加更完整的安全基线，包括密钥轮换、端口暴露、CORS、审计和备份。

## 跟踪方式

- 当前版本计划记录在 `docs/todo/26.1.1-开发计划.md`。
- 每完成一个功能，需要更新任务状态、验证结果和遗留事项。
- 新版本发布时，在 `docs/todo/` 下按版本号新增开发计划文件。
