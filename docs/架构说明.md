# 架构说明

BuildingAI 是一个基于 pnpm workspace 和 Turbo 的单仓库项目，包含后端 API、前端客户端、CLI、核心库、AI SDK、扩展和技能体系。

## 仓库结构

- `packages/api`: NestJS 后端服务。
- `packages/client`: React + Vite 前端客户端，并包含 Tauri 桌面打包能力。
- `packages/cli`: 启动、部署、PM2、扩展相关 CLI。
- `packages/core`: 核心 Nest 模块与通用能力。
- `packages/@buildingai`: 共享包，包括 `db`、`cache`、`config`、`ai-sdk`、`logger`、`utils`、`web` 等。
- `extensions`: 扩展应用目录。
- `skills`: 项目内置技能目录。
- `public`: 前端构建后的静态资源目录。

## 后端入口

后端入口是 `packages/api/src/main.ts`。启动流程包括：

1. 加载环境变量。
2. 调用 `AppModule.register()` 创建动态模块。
3. 创建 Nest Express 应用。
4. 注册 JSON、URL encoded、XML body parser 和 cookie parser。
5. 根据环境变量启用 CORS。
6. 注册静态资源目录。
7. 注册全局 `ValidationPipe`、响应转换拦截器、HTTP 日志拦截器和异常过滤器。
8. 监听 `SERVER_PORT`，默认 `4090`。

根模块位于 `packages/api/src/modules/app.module.ts`，负责装配系统模块、AI 模块、数据库、缓存、权限守卫和扩展模块。

## API 路由

控制器装饰器位于 `packages/api/src/common/decorators/controller.decorator.ts`。

- `@ConsoleController`: 后台管理 API，默认前缀 `consoleapi`。
- `@WebController`: 前台 Web API，默认前缀 `api`。
- `@OpenApiController`: 开放 API，固定前缀 `/v1`。

前缀可通过 `.env` 中的 `VITE_APP_WEB_API_PREFIX` 和 `VITE_APP_CONSOLE_API_PREFIX` 调整。

## 认证与权限

全局守卫在 `AppModule` 中通过 `APP_GUARD` 注册：

- `DemoGuard`: 演示环境限制。
- `AuthGuard`: JWT 登录鉴权。
- `AgentGuard`: Agent API Key 和公开访问身份解析。
- `ExtensionGuard`: 扩展路由合法性校验。
- `PermissionsGuard`: 后台权限校验。
- `SuperAdminGuard`: 超级管理员限制。
- `MemberOnlyGuard`: 会员功能限制。

JWT token 会落库到用户 token 表，并结合内存缓存和 Redis 缓存提升校验效率。

## 数据库、缓存与队列

- 数据库使用 PostgreSQL + TypeORM。
- 数据库配置来自 `DB_*` 环境变量。
- TypeORM 命名策略使用蛇形命名。
- 系统实体集中在 `packages/@buildingai/db/src/entities`。
- 数据库模块会动态加载系统实体和扩展实体。
- Redis 由 `@buildingai/cache` 封装，支持 KV、Hash、Pub/Sub、分布式锁和健康检查。
- 队列基于 BullMQ + Redis，默认包含 `default`、`email`、`vectorization`。

首次安装时，数据库初始化服务会同步 schema、执行种子数据；已安装环境会执行版本升级和扩展升级。

## AI 模块

AI 聚合模块位于 `packages/api/src/modules/ai/ai.module.ts`，包含：

- `agents`: Agent 创建、配置、聊天、发布、标注。
- `chat`: 通用聊天。
- `datasets`: 知识库、文档、分段、检索、向量化。
- `mcp`: MCP 服务和工具管理。
- `memory`: 用户与 Agent 记忆。
- `model`: 模型管理。
- `provider`: 服务商管理。
- `secret`: 密钥管理。

Agent 对话核心位于 `agent-chat-completion.service.ts`，支持模型路由、系统 Prompt、知识库检索工具、MCP 工具、附件读取、规划工具、快捷命令、标注回复、长期记忆、工具审批、计费和追问建议。

## 知识库

知识库模块位于 `packages/api/src/modules/ai/datasets`。

文档处理链路：

1. 上传或导入文档。
2. 解析文件内容。
3. 分段服务生成 segments。
4. `VectorizationTriggerService` 投递 `vectorization` 队列。
5. `VectorizationProcessor` 消费任务。
6. `VectorizationRunnerService` 调用 embedding 模型。
7. 写回分段向量和处理状态。

检索支持 `vector`、`fullText`、`hybrid`、`keyword`，并支持 topK、阈值、权重和 rerank。

## 模型与 Provider

Provider 和模型通过后台配置管理。调用模型时，业务服务根据 `AiModel.provider` 找到 Provider，再通过 Secret 服务读取密钥，最后调用 `@buildingai/ai-sdk`。

内置 Provider 适配包括 OpenAI、DeepSeek、智谱、Moonshot、SiliconFlow、通义、火山、混元、文心、星火、Ollama、MiniMax、Anthropic、Google、Cohere、OpenRouter、Azure 和 Custom 等。

## 扩展机制

扩展启动流程：

1. 读取 `extensions` 目录。
2. 根据扩展配置筛选启用扩展。
3. 初始化扩展缓存。
4. 为每个扩展创建独立 PostgreSQL schema。
5. 动态加载扩展 `build/index.js` 中导出的 `AppModule`。
6. 执行扩展 seed 和会员功能扫描同步。

扩展实体会从 `extension.path/build/db/entities/**/*.entity.js` 动态加载。

## 前端架构

前端客户端位于 `packages/client`，技术栈包括 React 19、TypeScript、Vite、React Router、TanStack Query、Zustand、Tailwind CSS、lucide-react 和 Tauri。

入口文件为 `packages/client/src/main.tsx`，挂载顺序包括：

1. `I18nProvider`
2. `QueryClientProvider`
3. `ThemeProvider`
4. `TooltipProvider`
5. `AlertDialogProvider`
6. `SettingsDialogProvider`
7. `Toaster`
8. `RouterProvider`

总路由位于 `packages/client/src/router/index.tsx`。

主要路由：

- `/install`: 初始化安装。
- `/login`: 登录。
- `/`: 动态首页和聊天入口。
- `/chat`、`/chat/:id`: 通用聊天。
- `/apps`: 应用广场。
- `/agents`: Agent 广场。
- `/agents/workspace`: 我的 Agent 工作区。
- `/agents/:id/configuration`: Agent 编排配置。
- `/agents/:id/chat`: Agent 聊天。
- `/agents/:id/publish`: Agent 发布。
- `/datasets`、`/datasets/:id`: 知识库。
- `/console/*`: 后台控制台。

## HTTP 服务封装

Web 侧共享包位于 `packages/@buildingai/web`。

- `@buildingai/http`: HTTP 基础客户端、响应解析、拦截器。
- `@buildingai/services`: 前台、控制台、共享 API 服务封装。
- `@buildingai/stores`: 登录态和全局状态。
- `@buildingai/ui`: UI 组件、布局、鉴权组件。

HTTP 响应标准结构为 `code/message/data/timestamp`，`code` 在 `20000-29999` 视为成功。请求会自动注入 `Authorization: Bearer <token>`。

## 健康检查

健康检查模块位于 `packages/api/src/modules/health`。

- `GET /consoleapi/health`: 基础健康检查。
- `GET /consoleapi/health/http`: HTTP ping。
- `GET /consoleapi/health/detail`: HTTP、内存、磁盘、数据库、Redis。
- `GET /consoleapi/health/db`: 数据库和 Redis。
