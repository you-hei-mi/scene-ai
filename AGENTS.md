# AGENTS.md

## 项目协作约定

- 所有对话、授权确认、进度说明、总结和文档内容默认使用中文。
- 开始开发前先阅读项目结构、根配置、相关包的 `package.json`、入口文件和已有实现，不凭空假设。
- 搜索文件和文本优先使用 `rg` 或 `rg --files`；没有 `rg` 时再使用 PowerShell 原生命令。
- 修改文件前先说明将要编辑的范围；不要回滚或覆盖他人未明确要求回滚的改动。
- 每完成一个功能或文档任务，必须同步更新 `docs/todo/26.1.1-开发计划.md` 的状态、验证结果和遗留事项。

## 多代理协作

- 可以使用 Task/子代理并行处理独立任务，但每个子代理必须拥有明确且互不重叠的文件或模块范围。
- 子代理只处理自己负责的文件，不修改其他代理或用户正在处理的文件。
- 子代理完成后需要汇报变更路径、验证方式和未完成事项，由主代理统一整合。
- 如果并行任务可能写入同一文件，先拆分章节草稿，再由主代理单点合并。

## 文档规范

- 文档统一放在根目录 `docs/`，版本计划统一放在 `docs/todo/`。
- Markdown 文件使用 UTF-8 编码，标题层级清晰，命令使用代码块或行内代码。
- 文档应基于源码、配置和实际脚本编写，不直接复制当前乱码中文 README 内容。
- 涉及部署、调试、数据库、Redis、密钥和公网端口时，必须区分开发环境与生产环境风险。

## 验证要求

- 代码改动优先运行相关测试、类型检查或构建。
- 仅 Markdown 文档改动时，可记录“未涉及代码，无需构建”，但仍需检查新增文件清单。
- 交付前执行 `git status --short`，确认新增或修改范围符合任务要求。
